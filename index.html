Fit WorkFlow

Что такое?

Git WorkFlow это практика, упрощающая работу над проектом.

Для чего?

Если его нет, то разработка идет хаотичным чередом. Команда не понимает где свой код, а где чужой.

как работает?

У нас есть основная ветка main, после мы создаем новую ветку develop, остальные разработчики создают изменения в ветке develop. Там то что они создали проходят тестирования и после заливаются в ветку main




Git rebase

Что такое?

команда в Git, позволяющая транспортировать изменения из одной ветки в другую.

Для чего?

Словно git merge, но для ленивых. Скрины из коммитов просто переносятся на ту ветку, которую хотите перенести

как работает?

У нас есть Git "переносит" твои коммиты и добавляет их поверх последних изменений из главной ветки. Это делает так, что история изменений выглядит как будто ты начал свою работу на самом свежем состоянии главной ветки, а не с устаревшими данными




Git fetch

Что такое?

команда, которая загружает изменения из удаленного репозитория.

Для чего?

используется, чтобы синхронизировать ваш локальный репозиторий с удаленным, чтобы увидеть изменения, сделанные другими разработчиками

как работает?

У нас есть Git обращается к удаленному репозиторию (например, на GitHub) и загружает все новые данные. После этого они сохраняются в вашем локальном репозитории, но ваша текущая ветка не изменяется
